Desktop Zones

  • KPI strip: today orders, waiting quote, unpaid balance, ready pickup
  • Priority queue: blocked, overdue, ready, and needs-customer-response cards
  • Recent orders: dense list with customer, device, stage, payment, and next action
  • Quick modules: buyback, inventory scan, message templates, and settings shortcuts

Mobile Zones

  • Floating header with store/day context and primary add action
  • Horizontal metric strip that fits 3-4 cards without wrapping
  • Priority business cards with status, amount, and next action
  • Recent activity cards using the mobile order-detail information hierarchy

Primary Actions

  • Create order
  • Open priority queue
  • Start scan
  • Send follow-up

States

  • loading skeleton
  • partial data warning
  • empty recent orders
  • selected priority

Motion

  • KPI click filters the queue with motion/fast chip feedback.
  • Priority card opens context rail or sheet with motion/standard.
  • Partial-data warning resolves with motion/instant badge and toast.

Acceptance

  • No hero or decorative chart dominates the first viewport.
  • Desktop warnings and next actions are visible without page navigation.
  • Mobile card hierarchy matches the protected order-detail standard.

Prototype Flows

Prototype

Desktop list selection

Trigger: Click dense row

Motion: motion/standard opacity + translateX 12

Right context panel opens without leaving the list; selected row and toolbar batch state update.
Prototype

Desktop filter refinement

Trigger: Open filter or segmented control

Motion: motion/fast opacity + y 4

Filter panel expands inside the toolbar area; table keeps layout width stable.
Prototype

Mobile card detail

Trigger: Tap business card

Motion: motion/standard sheet slide + opacity

Detail sheet uses floating-header hierarchy and keeps bottom actions visible.
Prototype

Mutation feedback

Trigger: Save, status change, send, approve, or import action

Motion: motion/instant button loading, then motion/fast toast

Button disables during pending; inline error or success toast appears without shifting layout.
Prototype

Skeleton to content

Trigger: Initial load or panel refresh

Motion: motion/slow shimmer fade to content

Skeleton occupies the final layout dimensions so dense pages do not jump.
